# Retro Skill — Sprint & Incident Retrospective Facilitator for EMs, PMs, SREs

> **Run a retro that produces action, not vibes.** Blameless, time-boxed, structured — every "what didn't go well" ends with an owner and a due date. Works for sprint retros and post-incident retros.

**Keywords**: sprint retro, incident retro, blameless retrospective, retrospective facilitator, ai retro, action items with owners, retro claude code skill

## Install

```bash
curl -sSL https://raw.githubusercontent.com/Sanmanchekar/skills-library/main/install.sh | bash -s -- retro
```

## What it does

- **Blameless by design** — behavior is a function of system + incentives, not people
- **4-round structure** — what went well → didn't → themes → actions
- **Post-incident variant** — adds `detect / prevent / respond / mitigate` categories
- Every action requires a **verb + specific artifact + owner + date** — blocks "we should communicate better"
- **Revisits prior-retro actions** at the start of the next retro — actions can't rot
- Facilitator rules: **time-box, everyone speaks, no solutioning in the diagnosis round**
- Produces a **written summary** — retro without notes = retro that never happened
